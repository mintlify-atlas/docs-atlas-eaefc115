---
title: "Extract Brand DNA from Your Website"
description: "Run /ads dna &lt;url&gt; to capture screenshots and extract colors, typography, voice, and imagery style into brand-profile.json for consistent ad creative."
---

`/ads dna <url>` is the first step of the creative pipeline. It fetches up to three pages from your website, captures screenshots of each, and analyzes the HTML, CSS, and Open Graph metadata to extract a structured brand identity profile. The result is `brand-profile.json` — a machine-readable file that every downstream command reads to stay on brand.

<Note>
  The URL is fetched by Claude Code running on your local machine. No data is sent to external servers beyond the normal fetch request to the target website.
</Note>

## Usage

```bash
/ads dna https://example.com
```

```bash
# Fast mode: homepage only, no screenshots
/ads dna https://example.com --quick
```

## What it captures

The command fetches pages in this order:

1. **Homepage** — dominant colors, hero headline tone, primary typography
2. **About page** — brand values, voice descriptors (`/about`, `/about-us`, `/our-story`)
3. **Product or services page** — imagery style, composition, subject matter (`/product`, `/products`, `/services`)

If a secondary page returns a 404, the extraction continues with fewer pages and notes lower confidence in the output.

### Screenshots

After fetching each page, the command captures three screenshots using `scripts/capture_screenshot.py` and saves them to `./brand-screenshots/`. These screenshots serve as visual style references when `/ads generate` produces images — the `visual-designer` agent passes the most relevant screenshot as a style anchor to the image generation API.

```
./brand-screenshots/
  {domain}_homepage.png
  {domain}_product.png
  {domain}_about.png
```

<Tip>
  If Playwright is not installed, screenshot capture is skipped gracefully. Install it with `python3 -m playwright install chromium` to enable it.
</Tip>

## What gets extracted

### Colors

The extractor reads CSS `background-color` and `color` properties on `body`, `header`, `.hero`, `h1`, `h2`, and `.btn` selectors. It also analyzes the dominant colors in the `og:image` meta tag. The result is four color roles:

| Field | What it captures |
|-------|-----------------|
| `primary` | Most prominent brand color |
| `secondary` | Supporting palette (array) |
| `background` | Page background color |
| `text` | Primary text color |
| `forbidden` | Colors explicitly excluded from generation prompts |

### Typography

Google Fonts `@import` URLs are parsed to extract font names. CSS `font-family` on `h1`, `h2`, and `body` is used as a fallback. If no fonts are detected, `heading_font` defaults to `null` and `body_font` to `"system-ui"`.

### Voice axes

The hero headline, subheadline, About page intro, and CTA text are analyzed against six axes on a 1–10 scale:

| Axis | 1 (left) | 10 (right) |
|------|----------|------------|
| `formal_casual` | Very corporate | Very conversational |
| `rational_emotional` | Data-driven | Emotionally evocative |
| `playful_serious` | Fun, humorous | Serious, no-nonsense |
| `bold_subtle` | Big claims, loud | Understated |
| `traditional_innovative` | Classic, established | Cutting-edge |
| `expert_accessible` | Deep jargon | Everyone understands |

Each axis also gets a confidence rating (High / Medium / Low) based on how many signals were found.

### Imagery style and aesthetic

From the `og:image` and visible hero images, the extractor identifies:
- Style: `professional photography`, `illustration`, `flat design`, or `mixed`
- Subjects: what appears in the imagery
- Composition: clean/minimal vs. busy/editorial
- Forbidden elements: inferred from brand positioning (e.g., enterprise brands get "cheesy stock photos" added to the forbidden list)

## Output: brand-profile.json

The file is written to `./brand-profile.json` in the current working directory. The schema follows the canonical brand DNA template:

```json
{
  "schema_version": "1.0",
  "brand_name": "Acme",
  "website_url": "https://acme.com",
  "extracted_at": "2026-04-25T10:00:00Z",
  "voice": {
    "formal_casual": 7,
    "rational_emotional": 4,
    "playful_serious": 5,
    "bold_subtle": 8,
    "traditional_innovative": 6,
    "expert_accessible": 5,
    "descriptors": ["authoritative", "warm", "direct"]
  },
  "colors": {
    "primary": "#1A2E4A",
    "secondary": ["#F4A623", "#FFFFFF"],
    "forbidden": ["#FF0000"],
    "background": "#F9F9F9",
    "text": "#1A1A1A"
  },
  "typography": {
    "heading_font": "Inter",
    "body_font": "Source Sans Pro",
    "pairing_descriptor": "modern sans-serif, clean and readable"
  },
  "imagery": {
    "style": "professional photography",
    "subjects": ["people using product", "clean product shots"],
    "composition": "clean backgrounds, good negative space",
    "forbidden": ["stock photo clichés", "corporate handshakes"]
  },
  "aesthetic": {
    "mood_keywords": ["trustworthy", "modern", "approachable"],
    "texture": "minimal",
    "negative_space": "generous"
  },
  "brand_values": ["transparency", "innovation", "customer-first"],
  "target_audience": {
    "age_range": "28-45",
    "profession": "marketing managers and small business owners",
    "pain_points": ["time-consuming reporting", "unclear ROI"],
    "aspirations": ["grow efficiently", "look professional"]

    "product": "./brand-screenshots/acme_product.png",
    "about": "./brand-screenshots/acme_about.png"
  }
}
```

Fields that cannot be extracted confidently are set to `null` — the extractor never guesses.

## How the profile feeds downstream steps

Once `brand-profile.json` exists, the rest of the pipeline reads it automatically:

- **`/ads create`** reads voice axes to calibrate copy tone, and mood keywords to inform concept direction
- **`/ads generate`** injects `colors.primary`, `aesthetic.mood_keywords`, and `imagery.forbidden` into every image generation prompt
- **`/ads photoshoot`** uses `colors.primary` for background accents and `target_audience` for lifestyle context

## Limitations

<Warning>
  Sites with fewer than 200 words of visible body text produce lower-confidence profiles. JavaScript-heavy SPAs (React, Next.js with SSR disabled) may have no extractable CSS. In both cases the extractor notes the limitation in a field comment and continues rather than failing.
</Warning>

| Limitation | Behavior |
|-----------|----------|
| Sparse content (&lt;200 words) | Notes "Low confidence extraction" in output |
| JS-rendered SPA | Falls back to `og:image` for color extraction |
| Multi-brand enterprise | Run separately for each brand or sub-brand URL |
| Dark mode site | Swaps `background` and `text` values automatically |
| CSS-in-JS (styled-components, Tailwind JIT) | Uses `og:image` colors as fallback |
